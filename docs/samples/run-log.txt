# Shortened run log

Exact timings and row counts vary. The important properties are visible batch
boundaries, per-statement timing, and failure behavior controlled by
`ON_ERROR_STOP`.

If a statement fails, `psql -v ON_ERROR_STOP=1` exits. The current open
transaction is rolled back by PostgreSQL connection cleanup, while previously
committed batches remain committed.
